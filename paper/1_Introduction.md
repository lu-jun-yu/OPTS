强化学习领域长期存在两条几乎平行发展的技术路线。第一条路线是 `Search + Policy Projection`：典型代表是 AlphaGo Zero、AlphaZero 与 MuZero。这一类方法在每次决策前进行大量搜索，再将更强的搜索策略投影回神经网络。第二条路线是 `Rollout + Policy Gradient`：典型代表是 PPO 以及近年的 LLM 强化学习方法，如 GRPO、DAPO、REINFORCE++ 与 GPG。它们通过大量 rollout 收集奖励信号，并直接用策略梯度更新模型。前者擅长利用搜索强化局部决策，后者擅长在大规模动作空间与连续动作空间中进行端到端优化。

问题在于，这两条路线之间仍存在明显断层。搜索之所以强大，是因为它利用了后验信息来修正先验决策；但一旦把这种后验修正过的轨迹拿来训练策略网络，就会立刻遇到三个困难。

第一，`distribution shift`。如果搜索改变了采样分布，那么训练数据就不再严格来自当前策略，直接使用 on-policy 策略梯度会引入偏差，而强行使用重要性采样往往又会遭遇估计不准和方差爆炸。

第二，`parallelism mismatch`。经典 MCTS 依赖节点访问计数、逐次扩展与细粒度回传，这类计算图天然偏串行，与现代 GPU 友好的大批量 rollout 框架并不匹配。

第三，`objective mismatch`。策略投影方法需要一个“更强的下一步动作分布”，因此天然适合宽度优先的搜索；而策略梯度方法并不拟合完整动作分布，它更关心哪些轨迹片段带来了更高回报，因此它真正需要的是一种能够在轨迹深处持续发现高价值分支点的、深度式的搜索。

本文的出发点正是：如果我们想把搜索重新引入策略梯度训练，那么搜索本身也必须为策略梯度重新设计。它必须满足三个条件：`on-policy`、`parallel-friendly`、`depth-wise expansion`。围绕这三个条件，我们提出 `OPTS-TTPO` 框架。

其中，`TTPO (Tree Trajectory Policy Optimization)` 负责学习。它将链式轨迹上的策略梯度定理推广到同策略树轨迹：对每一个状态-动作对，我们不再简单平均所有出现次数，而是按照其祖先分支数乘积的倒数进行加权。进一步地，我们把 GAE 推广为 TreeGAE，使优势回传能够在树轨迹上递归进行，并在实现层面落实为 branch-weight 修正的策略损失、价值损失与优势归一化。

`OPTS (On-Policy Tree Search)` 负责搜索。它不尝试构建完整的访问统计树，而是先在已有树轨迹中抽取一条当前最优路径，再沿这条路径计算 TUCT 分数，从中选择一个最值得重新采样的分支点。这样做保留了“搜索强化局部决策”的能力，同时避免了传统树搜索对串行结构和全动作展开的依赖。

更重要的是，不同任务的预算定义不同，算法也必须随之调整。在 Atari 和 MuJoCo 中，预算消耗发生在动作级别，越深的重新搜索意味着额外占用更多 action budget，因此开发项需要带长度惩罚；在 LLM 中，预算通常按整条回答或整个 episode 计量，同样一次补采样并不会因为更深而额外消耗独立预算，因此开发项不应带长度惩罚。本文将这一差异直接写入算法定义，而不是把所有场景硬塞进同一个公式。

本文的贡献可以概括为三点。

1. 我们提出 TTPO，将树轨迹上的无偏策略梯度、TreeGAE、加权优势归一化以及带 branch weight 修正的 PPO/value 损失统一到同一个实现框架中。
2. 我们提出 OPTS，一种专门面向策略梯度训练的并行同策略树搜索方法；其核心是预算感知的 TUCT 选点准则以及“从选中节点的父节点重新分支”的实现方式。
3. 我们基于当前仓库中可核实的代码，给出 Atari、MuJoCo 与 LLM 三类场景下的一致化实例化，并整理出一版可用于 NeurIPS 2026 投稿继续打磨的论文初稿。现有结果显示，该方向具有明显潜力，但不同任务上的收益形态并不完全一致，这恰恰说明预算定义、搜索深度与优化目标的耦合关系值得被系统研究。
